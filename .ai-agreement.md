# AI Development Agreement

## Introduction
This agreement defines the scope of AI work involved in the development of this project. Prioritizing security and confidentiality of sensitive data.

## Restricted File Access and Modification Requirements

### 1. Environment Variables Files (Strictly Forbidden)
- **`.env`** - Absolutely forbidden to read, modify, or change.
- **`.env.local`** - Absolutely forbidden to read, modify, or change.
- **`.env.development`** - Absolutely forbidden to read, modify, or change.
- **`.env.production`** - Absolutely forbidden to read, modify, or change.

### 2. Critical Configuration Files
- **`package-lock.json`** - Absolutely forbidden to modify directly.
- **`yarn.lock`** - Absolutely forbidden to modify directly.
- **`.git/config`** - Absolutely forbidden to access or modify.
- **`.gitignore`** - Can only be modified with permission.

### 3. Security Files
- **All types of files containing Secret Keys**
- **SSL Certificate files** (`*.crt`, `*.key`, `*.pem`)
- **SSH files** Keys** (`id_rsa`, `id_ed25519`)

### 4. Folders that cannot be modified:
- **`node_modules/`** - Do not modify files within this folder.
- **`dist/`** or **`build/`** - Do not modify files that have already been built.
- **`.git/`** - Do not access or modify.

## AI Workflow Rules

### ✅ What is allowed:
1. Create new files in the appropriate folders.
2. Modify source code files (`*.js`, `*.ts`, `*.jsx`, `*.tsx`, `*.py`, `*.html`, `*.css`).
3. Suggest package installation via `npm install` or `yarn add`.
4. Create or modify files. documentation (`*.md`)
5. Edit safe configuration files (`package.json`, `tsconfig.json`)

### ❌ Things to avoid:
1. **Do not modify or create a new `.env` file.**
2. **Do not display or reveal values ​​in a `.env` file.**
3. **Do not commit `.env` files to the repository.**
4. **Do not modify files directly in `node_modules/`.**
5. **Do not modify files in `.git/`.**
6. **Do not modify files that have already been built in `dist/` or `build/`.**

## Managing Environment Variables

### Creating a `.env.example` file
It is allowed to create or edit a **`.env.example` file** to show an example of the structure of necessary environment variables. It must not contain actual values:

```env
# .env.example - Example only. Do not put actual values.
DATABASE_URL=your_database_url_here
API_KEY=your_api_key_here
JWT_SECRET=your_jwt_secret_here
